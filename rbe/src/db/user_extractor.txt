This is a very good next step — the current_user extractor is basically what turns your Axum app from “toy API” into a real backend.

I’ll show you a clean, production-style version using Axum + SQLx + JWT.

🧠 Goal

You want this in handlers:

async fn me(CurrentUser(user): CurrentUser) -> String {
    format!("Hello {}", user.email)
}

Instead of manually parsing tokens everywhere.

We need 3 parts:

JWT middleware logic (extract token)
DB user loading
Axum extractor (FromRequestParts)

⚙️ CurrentUser extractor

This is the core:

use axum::{
    async_trait,
    extract::FromRequestParts,
    http::{request::Parts, StatusCode},
};
use jsonwebtoken::{decode, DecodingKey, Validation, Algorithm};

use crate::{AppState, User, Claims};

pub struct CurrentUser(pub User);

#[async_trait]
impl FromRequestParts<AppState> for CurrentUser {
    type Rejection = StatusCode;

    async fn from_request_parts(
        parts: &mut Parts,
        state: &AppState,
    ) -> Result<Self, Self::Rejection> {

        // 1. Get Authorization header
        let auth_header = parts
            .headers
            .get("authorization")
            .and_then(|h| h.to_str().ok())
            .ok_or(StatusCode::UNAUTHORIZED)?;

        // 2. Extract token
        let token = auth_header
            .strip_prefix("Bearer ")
            .ok_or(StatusCode::UNAUTHORIZED)?;

        // 3. Decode JWT
        let token_data = decode::<Claims>(
            token,
            &DecodingKey::from_secret(state.jwt_secret.as_bytes()),
            &Validation::new(Algorithm::HS256),
        )
        .map_err(|_| StatusCode::UNAUTHORIZED)?;

        let user_id = token_data.claims.sub;

        // 4. Load user from DB
        let user = sqlx::query_as!(
            User,
            r#"
            SELECT id, email
            FROM users
            WHERE id = $1
            "#,
            user_id
        )
        .fetch_one(&state.db)
        .await
        .map_err(|_| StatusCode::UNAUTHORIZED)?;

        Ok(CurrentUser(user))
    }
}
🌐 Usage in handlers
use axum::extract::State;

pub async fn me(
    CurrentUser(user): CurrentUser,
    State(_state): State<AppState>,
) -> String {
    format!("Hello {}", user.email)
}
🧩 Router setup
use axum::{Router, routing::get};

pub fn router(state: AppState) -> Router {
    Router::new()
        .route("/me", get(me))
        .with_state(state)
}
🔥 What you gained

Now every request:

✔ automatically extracts JWT
✔ validates token
✔ loads user from DB
✔ gives you typed User

No boilerplate in handlers.

🧠 Real-world upgrade options
1. Add roles to extractor
pub struct CurrentUser {
    pub user: User,
    pub roles: Vec<String>,
}

Then load roles in same query.

2. Cache user (important for performance)

Instead of DB hit every request:

Redis
in-memory cache
token-embedded claims (lightweight apps)
3. Better rejection type

Instead of StatusCode, use:

Json<ApiError>
4. Split extractor logic

Production pattern:

auth/
  jwt.rs
  extractor.rs
  service.rs
🚀 Mental model (important)

Axum extractor = middleware + dependency injection

Request
